The review focuses on policies, procedures, and access control mechanisms that delineate and enforce access restrictions, thereby safeguarding cardholder data from unauthorized access, especially in multi-tenant environments. This process is crucial for maintaining the confidentiality and integrity of cardholder data as stipulated in PCI DSS v4.0 Requirement A1.1.2.a.

**Category**: System Evidence

**Reference Name**: Requirement A1.1.2.b System Configuration Customer Access Review

**Details**: 
- **System Configuration Customer Access Review**: This evidence consists of examining system configurations to verify that customers' privileges are correctly established, allowing access solely to their own account data and Cardholder Data Environment (CDE). The focus of this review is on ensuring that system settings and access controls are accurately configured to maintain strict boundaries between different customers' data. This verification is essential for compliance with PCI DSS v4.0 Requirement A1.1.2.b, playing a critical role in preserving the security and segregation of sensitive cardholder information in multi-tenant environments.

### sub-appendix a1.1.3

**Category**: Documentation Evidence

**Reference Name**: Requirement A1.1.3 Customer Privileges Allocation Verification

- **Customer Privileges Allocation Verification**: This evidence involves the examination of customer privileges to ensure that each customer has access only to the resources allocated to them.
